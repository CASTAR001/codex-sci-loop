# Audit Directory

Each phase gets one subdirectory:

```text
audits/<phase-id>/
  audit-input.md
  audit.md
```

`audit-input.md` is generated by `Prepare-LoopAuditPackage.ps1`.

`audit.md` is written by Codex after inspecting:

- `.ai-loop/status.json`
- phase prompt
- Kimi Worker report
- diff patch
- verification log
- repository status
- relevant source files changed by the phase

The audit must end with exactly one decision: `ACCEPTED`, `REWORK`, or
`BLOCKED`.

Use `.ai-loop/templates/audit-input.md` and `.ai-loop/templates/audit.md` as the
human-readable templates for those files.
