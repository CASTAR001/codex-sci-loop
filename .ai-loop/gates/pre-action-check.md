# Pre-Action Check

Before planning or modifying files, answer these questions:

1. Have I read `.ai-loop/README.md`, `activeContext.md`,
   `constraint-ledger.md`, and this file?
2. Am I about to modify governance files? If yes, did the user explicitly enter
   a harness maintenance phase?
3. Does this action violate any hard constraint?
4. Does this action repeat a known failure mode from `failure-ledger.md`?
5. Is a scientific correctness skill required for this task?
6. What evidence will prove completion?
7. What files must be inspected before editing?
8. What is the stop condition?

If any answer is unclear, stop and ask or mark `BLOCKED`.
